### 3.4 问题3：围栏状态切换接口路径疑似错误

**模块名称**：围栏状态切换 (Fence Status Toggle)
**文件位置**：`NewFrontend/js/services/fences.js`
**当前路径**：`/api/v1/fences/${id}/status`
**预期路径**：`/api/v1/fences/{fence_id}/status`

**问题代码**：
```javascript
// NewFrontend/js/services/fences.js (line 24-26)
async toggleStatus(id) {
  const data = await request.patch(`/api/v1/fences/${id}/status`)  // ⚠️ 需验证
  return data
}
```

**后端实际路由**（`fences.py` 第78-85行）：
```python
@router.patch("/{fence_id}/status", response_model=ResponseBase[GeoFenceResponse])
def toggle_fence_status(
    fence_id: int,  # ⚠️ 参数名为 fence_id
    db: Session = Depends(get_db),
    current_user: User = Depends(require_admin),
):
    service = FenceService(db)
    fence = service.toggle_fence_status(fence_id)
    return ResponseBase(data=fence, message="状态切换成功")
```

**分析**：
- 前端使用模板字符串 `` `/api/v1/fences/${id}/status` ``
- 如果 `id` 变量值正确（如 `1`, `2` 等数字），则生成的路径为 `/api/v1/fences/1/status`
- 后端期望的路径为 `/api/v1/fences/{fence_id}/status`
- 两者在语义上是一致的，只要 `id` 值正确传递

**建议**：
- 确认前端调用时传递的参数名和值是否正确
- 建议统一使用 `fence_id` 作为参数名以提高代码可读性
