# 红色文化AR游览应用 - 安卓打包指南

本文档详细介绍如何将基于Vite构建的前端项目打包为可在安卓设备上安装运行的APK应用程序。文档基于实际构建过程中遇到的问题编写，包含完整的故障排除方案。

## 一、项目概述

本项目是一个红色文化AR游览前端应用，使用Vite作为构建工具，支持多个页面包括首页、登录、地图、人物、聊天、统计和个人中心等模块。项目采用模块化开发结构，配置了高德地图API集成。

### 1.1 技术栈分析

- **构建工具**：Vite 5.4.0
- **项目类型**：多页面HTML应用（非SPA）
- **UI框架**：原生JavaScript组件化开发
- **地图服务**：高德地图 Web API 2.0
- **目标平台**：Android（通过Capacitor打包）
- **Android构建工具**：Gradle 9.4.1 + Android Gradle Plugin 8.13.0
- **Java版本**：JDK 17

### 1.2 项目结构说明

```
NewFrontend/

## 二、环境配置要求

在开始打包之前，需要确保开发环境满足以下要求。

### 2.1 必需软件

#### 2.1.1 Node.js 环境

- **版本要求**：Node.js 18.x 或更高版本
- **推荐版本**：Node.js 20.x LTS
- **验证命令**：在终端执行 `node -v` 查看当前版本

#### 2.1.2 Java Development Kit (JDK)

- **版本要求**：JDK 17
- **安装路径示例**：`D:\Program Files\Java\jdk-17`
- **验证命令**：在终端执行 `java -version` 查看当前版本
- **重要说明**：Capacitor 7.x 默认要求 Java 21，但 JDK 17 也可以使用，需要修改配置文件（详见第四节）

> ⚠️ **关键提示**：如果系统未安装 JDK 或未配置 `JAVA_HOME` 环境变量，Gradle 构建将失败并报错：
> ```
> ERROR: JAVA_HOME is not set and no 'java' command could be found in your PATH.
> ```

#### 2.1.3 Gradle

- **版本要求**：Gradle 8.0 或更高版本
- **本项目使用版本**：Gradle 9.4.1
- **安装路径示例**：`D:\Users\Roaming\Gradle\gradle-9.4.1\bin`
- **说明**：可以使用 Gradle Wrapper（项目自带）或本地安装的 Gradle

#### 2.1.4 Android Studio

- **版本要求**：Android Studio 2022.0 或更高版本
- **本项目使用版本**：Android Studio Panda 4
- **必需组件**：
  - Android SDK Platform 36（Android 16）
  - Android SDK Build-Tools
  - Android SDK Command-line Tools
  - Android Emulator（用于模拟器测试）
- **SDK路径示例**：`C:\Users\admire\AppData\Local\Android\Sdk`
- **配置环境变量**：
  - `ANDROID_HOME`：指向Android SDK安装目录
  - `ANDROID_SDK_ROOT`：同ANDROID_HOME
  - 将 `platform-tools` 和 `cmdline-tools\latest\bin` 添加到系统PATH

### 2.2 环境变量配置

完成软件安装后，需要配置以下系统环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `JAVA_HOME` | `D:\Program Files\Java\jdk-17` | JDK安装路径 |
| `ANDROID_HOME` | `C:\Users\admire\AppData\Local\Android\Sdk` | Android SDK路径 |
| `ANDROID_SDK_ROOT` | `C:\Users\admire\AppData\Local\Android\Sdk` | 同ANDROID_HOME |
| `GRADLE_HOME` | `D:\Users\Roaming\Gradle\gradle-9.4.1` | Gradle安装路径（可选） |

**PATH 追加项**：

```
%JAVA_HOME%\bin
%ANDROID_HOME%\platform-tools
%ANDROID_HOME%\cmdline-tools\latest\bin
D:\Users\Roaming\Gradle\gradle-9.4.1\bin
```

**Windows 配置方法**：

1. 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
2. 在"系统变量"中新建或编辑上述变量
3. 编辑 `Path` 变量，添加上述追加项
4. 点击"确定"保存，重新打开终端使配置生效

**命令行快速配置（临时生效，仅当前终端会话）**：

```powershell
$env:JAVA_HOME = 'D:\Program Files\Java\jdk-17'
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
```

### 2.3 环境验证

完成安装后，可通过以下命令验证环境配置是否正确：

```powershell
# 验证Node.js版本
node -v

# 验证npm版本
npm -v

# 验证Java版本
java -version
# 预期输出：openjdk version "17.x.x" 或类似

# 验证Gradle版本
gradle -v
# 预期输出：Gradle 9.4.1

# 验证Android SDK配置
echo $env:ANDROID_HOME
# 预期输出：C:\Users\admire\AppData\Local\Android\Sdk

# 验证adb工具
adb version
```

## 三、Capacitor 打包方案

本项目采用 Capacitor 作为跨平台打包方案。Capacitor 是由 Ionic 团队开发的开源项目，可以将任何Web应用打包为原生移动应用，支持Android和iOS平台。

### 3.1 方案优势

- **原生体验**：提供接近原生应用的用户体验和性能
- **插件生态**：支持丰富的原生功能插件（相机、地理位置、推送通知等）
- **热更新**：支持通过Web目录更新应用内容
- **调试便捷**：支持使用Chrome DevTools直接调试移动端应用
- **社区活跃**：拥有活跃的社区支持和完善的官方文档

### 3.2 替代方案对比

| 特性 | Capacitor | WebView直接打包 | uni-app |
|------|-----------|----------------|---------|
| 学习成本 | 低 | 低 | 中 |
| 原生功能 | 丰富插件 | 有限 | 丰富 |
| 性能 | 优秀 | 良好 | 良好 |
| 调试体验 | Chrome直接调试 | 需特殊配置 | 需要配置 |
| 更新方式 | Web目录热更新 | 重新打包 | 支持云端更新 |

## 四、详细打包步骤

### 4.1 第一阶段：项目准备与依赖安装

#### 步骤1：进入项目目录

打开终端，进入前端项目目录：

```powershell
cd e:\Code\NewGaoDe\GaoDe\NewFrontend
```

#### 步骤2：安装基础依赖

首先安装项目已有的依赖（如果尚未安装）：

```powershell
npm install
```

这条命令会读取 `package.json` 中的依赖列表并安装。

#### 步骤3：安装 Capacitor 核心包

安装 Capacitor 的核心运行时和命令行工具：

```powershell
npm install @capacitor/core @capacitor/cli
```

- `@capacitor/core`：Capacitor的核心运行时库
- `@capacitor/cli`：Capacitor的命令行工具，用于初始化、添加平台、同步等操作

### 4.2 第二阶段：初始化 Capacitor

#### 步骤4：初始化 Capacitor 配置

执行初始化命令：

```powershell
npx cap init
```

执行后会提示输入以下信息：

- **App Name（应用名称）**：输入 `红色文化AR游览`
- **App ID（包名）**：输入包标识符，格式为 `com.organization.appname`，例如 `com.xy.ar`
- **Web Directory（Web目录）**：输入 `dist`（这是Vite构建输出的目录）

也可以使用非交互式方式直接指定参数：

```powershell
npx cap init "红色文化AR游览" "com.xy.ar" --web-dir=dist
```

初始化完成后，会在项目根目录生成 `capacitor.config.json` 配置文件。

#### 步骤5：安装 Android 平台支持

安装 Capacitor 的 Android 平台包：

```powershell
npm install @capacitor/android
```

#### 步骤6：添加 Android 平台

将 Android 平台添加到项目中：

```powershell
npx cap add android
```

执行此命令后，Capacitor会在项目中创建一个 `android` 目录，包含完整的Android Studio项目文件。

### 4.3 第三阶段：构建与同步

#### 步骤7：构建 Web 应用

将前端项目构建为静态文件：

```powershell
npm run build
```

Vite会读取 `vite.config.js` 中的配置，将项目构建输出到 `dist` 目录。构建产物包含所有HTML、CSS、JavaScript和静态资源文件。

#### 步骤8：同步 Web 内容到 Android 项目

将构建产物同步到Android项目中：

```powershell
npx cap copy android
```

此命令会将 `dist` 目录下的所有文件复制到 `android\app\src\main\assets\public` 目录。

#### 步骤9：验证同步结果

可以检查同步是否成功：

```powershell
# Windows系统
dir android\app\src\main\assets\public

# 或者使用Capacitor命令查看状态
npx cap ls
```

### 4.4 第四阶段：修改 Java 版本兼容性配置（关键步骤）

> ⚠️ **重要**：Capacitor 7.x 默认使用 Java 21，但本项目环境使用 JDK 17。如果不修改以下配置，构建将失败并报错：
> ```
> 错误: 无效的源发行版：21
> ```
> 必须修改以下 3 个文件，将 `JavaVersion.VERSION_21` 改为 `JavaVersion.VERSION_17`。

#### 步骤10：修改 capacitor-android 模块的 Java 版本

打开文件 `node_modules\@capacitor\android\capacitor\build.gradle`，找到 `compileOptions` 部分：

**修改前**：

```gradle
compileOptions {
    sourceCompatibility JavaVersion.VERSION_21
    targetCompatibility JavaVersion.VERSION_21
}
```

**修改后**：

```gradle
compileOptions {
    sourceCompatibility JavaVersion.VERSION_17
    targetCompatibility JavaVersion.VERSION_17
}
```

#### 步骤11：修改 app 模块的 Capacitor 构建配置

打开文件 `android\app\capacitor.build.gradle`，找到 `compileOptions` 部分：

**修改前**：

```gradle
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_21
      targetCompatibility JavaVersion.VERSION_21
  }
}
```

**修改后**：

```gradle
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_17
      targetCompatibility JavaVersion.VERSION_17
  }
}
```

#### 步骤12：修改 Cordova 插件模块的 Java 版本

打开文件 `android\capacitor-cordova-android-plugins\build.gradle`，找到 `compileOptions` 部分：

**修改前**：

```gradle
compileOptions {
    sourceCompatibility JavaVersion.VERSION_21
    targetCompatibility JavaVersion.VERSION_21
}
```

**修改后**：

```gradle
compileOptions {
    sourceCompatibility JavaVersion.VERSION_17
    targetCompatibility JavaVersion.VERSION_17
}
```

> 📝 **注意**：`capacitor.build.gradle` 和 `capacitor-cordova-android-plugins/build.gradle` 文件头部注释标明 "DO NOT EDIT THIS FILE! IT IS GENERATED EACH TIME 'capacitor update' IS RUN"，意味着每次执行 `npx cap sync` 或 `npx cap update` 后这些文件会被重新生成，需要重新修改。`node_modules` 下的文件在 `npm install` 后也会被重置。

### 4.5 第五阶段：配置 Android 项目

#### 步骤13：配置应用图标

将图标资源放置到以下目录：

```
android\app\src\main\res\
├── drawable-hdpi/     # 72x72 像素
├── drawable-mdpi/     # 48x48 像素
├── drawable-xhdpi/    # 96x96 像素
├── drawable-xxhdpi/   # 144x144 像素
└── drawable-xxxhdpi/  # 192x192 像素
```

将对应尺寸的PNG图标文件替换为 `ic_launcher.png` 即可。

#### 步骤14：配置应用名称

打开 `android\app\src\main\res\values\strings.xml` 文件，修改应用名称：

```xml
<resources>
    <string name="app_name">红色文化AR游览</string>
</resources>
```

#### 步骤15：配置网络权限（重要）

如果应用需要访问HTTP接口（如地图API、后台服务等），需要配置网络安全设置。

打开 `android\app\src\main\res\xml\network_security_config.xml` 文件（如果不存在则创建），添加以下内容：

```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

然后在 `AndroidManifest.xml` 的 `<application>` 标签中添加属性：

```xml
<application
    android:networkSecurityConfig="@xml/network_security_config"
    android:usesCleartextTraffic="true"
    ... >
```

#### 步骤16：配置高德地图权限

在 `AndroidManifest.xml` 中添加地图相关权限：

```xml
<!-- 位置权限 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- 网络权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

### 4.6 第六阶段：构建 APK

构建 APK 有两种方式：命令行构建和 Android Studio 构建。

#### 方式一：命令行构建（推荐）

命令行构建更可靠，不受 Android Studio 界面差异影响。

**步骤17a：使用本地 Gradle 构建**

```powershell
# 设置 JAVA_HOME（如果未配置系统环境变量）
$env:JAVA_HOME = 'D:\Program Files\Java\jdk-17'
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"

# 进入 Android 项目目录
cd e:\Code\NewGaoDe\GaoDe\NewFrontend\android

# 使用本地安装的 Gradle 构建
D:\Users\Roaming\Gradle\gradle-9.4.1\bin\gradle.bat assembleDebug
```

**步骤17b：使用 Gradle Wrapper 构建**

```powershell
# 设置 JAVA_HOME（如果未配置系统环境变量）
$env:JAVA_HOME = 'D:\Program Files\Java\jdk-17'
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"

# 进入 Android 项目目录
cd e:\Code\NewGaoDe\GaoDe\NewFrontend\android

# 使用 Gradle Wrapper 构建
.\gradlew.bat assembleDebug
```

> 📝 **说明**：`assembleDebug` 构建调试版APK，`assembleRelease` 构建正式版APK。首次构建需要下载依赖，耗时较长（约1-3分钟）。

**构建成功标志**：

```

97 actionable tasks: 97 executed
```

**构建产物位置**：

```
android\app\build\outputs\apk\debug\app-debug.apk
```

#### 方式二：Android Studio 构建

**步骤17c：在 Android Studio 中打开项目**

执行以下命令自动打开Android Studio：

```powershell
npx cap open android
```

或者手动打开Android Studio，选择 "Open an existing project"，然后选择项目中的 `android` 文件夹。

**步骤17d：配置 Gradle 和 JDK**

1. 打开 **File** → **Settings** → **Build, Execution, Deployment** → **Build Tools** → **Gradle**
2. 在 **Gradle used in this project** 下选择 **Specified location**
3. 设置路径为：`D:\Users\Roaming\Gradle\gradle-9.4.1`
4. 在 **Gradle JDK** 下拉框中选择 JDK 17（路径：`D:\Program Files\Java\jdk-17`）
5. 点击 **Apply** 和 **OK**

**步骤17e：同步并构建**

1. 等待 Gradle 同步完成（右下角进度条）
2. 同步成功后，在 Gradle 面板展开 **app** → **Tasks** → **build** → 双击 **assembleDebug**
3. 或点击菜单栏 **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**
4. 等待构建完成

> ⚠️ **Android Studio Panda 4 注意事项**：如果 Gradle 面板中没有显示 Tasks 列表，说明 Gradle 同步未成功。请先解决同步错误（常见问题见第五节），同步成功后 Tasks 会自动出现。

### 4.7 第七阶段：安装到设备进行测试

**方法一：通过USB安装到真机**

1. 使用USB数据线将安卓设备连接到电脑
2. 在设备上开启开发者模式（设置 → 关于手机 → 连续点击"版本号"7次）
3. 在设备上启用USB调试（设置 → 开发者选项 → USB调试）
4. 执行以下命令：

```powershell
adb install -r android\app\build\outputs\apk\debug\app-debug.apk
```

**方法二：通过模拟器安装**

1. 在Android Studio中创建并启动模拟器
2. 将APK拖拽到模拟器窗口中进行安装
3. 或执行命令：

```powershell
adb install -r android\app\build\outputs\apk\debug\app-debug.apk
```

**方法三：直接复制**

将 `app-debug.apk` 文件通过微信、QQ或数据线传输到手机，在手机上点击安装。

### 4.8 第八阶段：构建正式版（Release）APK

#### 步骤18：生成签名密钥

首次发布需要创建签名密钥。在命令行执行：

```powershell
keytool -genkey -v -keystore D:\keys\myapp.jks -keyalg RSA -keysize 2048 -validity 9125 -alias mykey
```

参数说明：

| 参数 | 说明 |
|------|------|
| `-keystore` | 密钥库文件保存路径 |
| `-keyalg RSA` | 使用RSA加密算法 |
| `-keysize 2048` | 密钥长度2048位 |
| `-validity 9125` | 有效期25年（9125天） |
| `-alias` | 密钥别名 |

#### 步骤19：配置签名信息

在 `android\app\build.gradle` 中添加签名配置：

```gradle
android {
    ...
    signingConfigs {
        release {
            storeFile file('D:\\keys\\myapp.jks')
            storePassword '你的密钥库密码'
            keyAlias 'mykey'
            keyPassword '你的密钥密码'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

#### 步骤20：构建 Release 版本

```powershell
cd e:\Code\NewGaoDe\GaoDe\NewFrontend\android
D:\Users\Roaming\Gradle\gradle-9.4.1\bin\gradle.bat assembleRelease
```

构建完成后，Release版APK位于：

```
android\app\build\outputs\apk\release\app-release.apk
```

## 五、常见问题解决方案

### 5.1 环境配置问题

#### 问题1：Gradle 下载失败或超时

**错误表现**：

```
Exception in thread "main" java.io.IOException: Downloading from https://services.gradle.org/distributions/gradle-8.14.3-all.zip failed: timeout (10000ms)
Caused by: java.net.SocketTimeoutException: Connect timed out
```

**原因分析**：

Gradle Wrapper 尝试从官方服务器下载指定版本的 Gradle，由于网络问题（国内访问 Gradle 官方服务器较慢）导致下载超时。

**解决方案**：

**方案A：使用本地安装的 Gradle（推荐）**

跳过 Gradle Wrapper，直接使用本地安装的 Gradle：

```powershell
cd e:\Code\NewGaoDe\GaoDe\NewFrontend\android
D:\Users\Roaming\Gradle\gradle-9.4.1\bin\gradle.bat assembleDebug
```

**方案B：删除损坏的缓存后重试**

1. 关闭 Android Studio
2. 删除损坏的 Gradle 缓存目录：

```powershell
# 打开缓存目录
explorer "$env:USERPROFILE\.gradle\wrapper\dists"

### 5.2 构建过程问题

#### 问题5：Java 版本不兼容（无效的源发行版：21）

**错误表现**：

```
Execution failed for task ':capacitor-android:compileDebugJavaWithJavac'.
> Java compilation initialization error
    错误: 无效的源发行版：21
```

或：

```
Execution failed for task ':app:compileDebugJavaWithJavac'.
> Java compilation initialization error
    错误: 无效的源发行版：21
```

**原因分析**：

Capacitor 7.x 的构建配置文件中硬编码了 `JavaVersion.VERSION_21`，但系统安装的是 JDK 17，无法编译 Java 21 的代码。

**解决方案**：

需要修改以下 3 个文件，将 `VERSION_21` 改为 `VERSION_17`：

| 文件路径 | 说明 |
|----------|------|
| `node_modules\@capacitor\android\capacitor\build.gradle` | Capacitor Android 核心库 |
| `android\app\capacitor.build.gradle` | 应用级 Capacitor 配置 |
| `android\capacitor-cordova-android-plugins\build.gradle` | Cordova 插件配置 |

每个文件中找到 `compileOptions` 块，修改：

```gradle
// 修改前
compileOptions {
    sourceCompatibility JavaVersion.VERSION_21
    targetCompatibility JavaVersion.VERSION_21
}

// 修改后
compileOptions {
    sourceCompatibility JavaVersion.VERSION_17
    targetCompatibility JavaVersion.VERSION_17
}
```

> ⚠️ **重要提醒**：这些文件在执行 `npm install` 或 `npx cap sync` 后会被重置，每次都需要重新修改。

#### 问题6：Android Studio 中 Gradle 面板没有 Tasks 列表

- 打开 Android Studio 后，左侧 Gradle 面板中看不到 Tasks 列表
- 找不到 Build APK(s) 选项

3. 解决后点击工具栏的 🐘 图标（Sync Project with Gradle Files）重新同步
4. 同步成功后，Gradle 面板会显示完整任务列表

**替代方案**：使用命令行构建，不依赖 Android Studio 界面：
