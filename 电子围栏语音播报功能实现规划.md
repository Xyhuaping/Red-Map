# 电子围栏语音播报功能实现规划

## 一、功能概述

当用户进入电子围栏区域触发围栏时，系统自动播放该围栏关联的语音内容，实现"进入景点区域即听到语音讲解"的沉浸式体验。

### 核心流程

```
用户进入围栏 → 心跳检测触发 → _handleTrigger() → 自动播放语音 + 弹窗/通知
```

### 支持两种语音来源

| 来源 | 说明 | 优先级 |
|------|------|--------|
| **自定义音频** | 管理员上传/配置 `audio_url`，触发时直接播放 | 高（优先使用） |
| **TTS 语音合成** | 无自定义音频时，使用浏览器 SpeechSynthesis API 将 `trigger_prompt` 文字转为语音 | 低（降级方案） |

### 阶段二：前端语音服务（2 项任务）

#### 任务 2.1：新建语音播报服务

**新建文件**: `NewFrontend/js/services/voice.js`

```javascript
class VoiceService {
  constructor() {
    this.currentAudio = null
    this.isSpeaking = false
  }

  // 主入口：根据围栏配置选择播放方式
  async play(fence) {
    if (!fence.enable_voice) return false

    this.stop()  // 停止上一次播放

    if (fence.audio_url) {
      return this._playAudio(fence.audio_url)
    } else if (fence.trigger_prompt) {
      return this._playTTS(fence.trigger_prompt)
    }
    return false
  }

  // 播放自定义音频文件
  async _playAudio(url) {
    return new Promise((resolve) => {
      const audio = new Audio(url)
      this.currentAudio = audio
      this.isSpeaking = true

      audio.onended = () => { this.isSpeaking = false; resolve(true) }
      audio.onerror = () => { this.isSpeaking = false; resolve(false) }
      audio.play().catch(() => { this.isSpeaking = false; resolve(false) })
    })
  }

  // TTS 降级方案：使用浏览器 SpeechSynthesis API
  async _playTTS(text) {
    if (!('speechSynthesis' in window)) return false

    this.isSpeaking = true
    return new Promise((resolve) => {
      const utterance = new SpeechSynthesisUtterance(text)
      utterance.lang = 'zh-CN'
      utterance.rate = 1.0
      utterance.onend = () => { this.isSpeaking = false; resolve(true) }
      utterance.onerror = () => { this.isSpeaking = false; resolve(false) }
      window.speechSynthesis.speak(utterance)
    })
  }

  // 停止播放
  stop() {
    if (this.currentAudio) {
      this.currentAudio.pause()
      this.currentAudio = null
    }
    if ('speechSynthesis' in window) {
      window.speechSynthesis.cancel()
    }
    this.isSpeaking = false
  }
}
```

**设计要点**：
- 自定义音频优先，TTS 降级
- 自动停止上一次播放，避免多个围栏语音叠加
- `play()` 返回 Promise，方便上层判断播放结果
- TTS 使用 `zh-CN` 中文语音

#### 任务 2.2：修改心跳触发逻辑

**文件**: `NewFrontend/js/services/heartbeat.js`

在 `_handleTrigger` 方法中新增语音播报调用：

```javascript
_handleTrigger(fence, figure) {
  this._recordTrigger(fence.id)

  // 新增：自动语音播报
  voiceService.play(fence)

  if (this._isAppForeground()) {
    this._showFencePopup(fence, figure)
  } else {
    this._showNotification(fence, figure)
  }

  this._recordTriggerToBackend(fence.id, figure?.id)
}
```

```javascript

```

## 六、注意事项

### 浏览器自动播放策略

现代浏览器（Chrome 71+、Safari、Firefox）限制了无用户交互的自动播放。由于围栏触发时用户正在使用 App（有交互），前台场景下自动播放通常可行。但需注意：

- **前台**：用户正在操作 App，`Audio.play()` 通常可自动播放
- **后台**：浏览器会阻止自动播放，需等用户点击通知回到前台后再播放
- **首次访问**：部分浏览器要求至少一次用户交互后才能自动播放音频

### Capacitor 原生环境

在 Capacitor 打包的 App 中，自动播放限制更宽松，`Audio()` 和 `SpeechSynthesis` 均可在前台自动播放。

### 音频文件格式

建议 `audio_url` 指向的音频文件使用以下格式：
- **MP3**：兼容性最好，推荐
- **AAC/M4A**：iOS 原生支持
- **OGG**：Android 原生支持

### 性能考虑

- 音频文件采用流式加载，无需等待全部下载完成
- TTS 使用浏览器原生 API，无额外网络请求
- 语音播报与弹窗/通知并行执行，不阻塞 UI
