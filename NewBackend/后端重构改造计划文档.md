## 二、重构目标与范围

### 2.1 核心目标

1. **安全加固**：修复全部安全隐患，建立完善的认证授权体系
2. **架构升级**：引入分层架构，实现业务逻辑与路由解耦
3. **功能完善**：实现后台管理系统，包含围栏可视化绘制功能
4. **前端兼容**：确保与前端改造计划（纯 HTML/CSS/JS + Fetch API）无缝对接
5. **生产就绪**：完善日志、监控、限流、异常处理等运维基础设施
6. **Ubuntu 适配**：确保系统在 Ubuntu Linux 环境下稳定运行

### 2.2 重构范围

| 范围 | 说明 | 改造程度 |
|------|------|---------|
| 认证授权 | 从 SHA256 升级为 bcrypt，增加 RBAC 权限体系 | 🔴 重构 |
| API 层 | 保持接口路径兼容，增加版本前缀和分页元数据 | 🟡 改造 |
| 业务逻辑 | 抽取 Service 层，路由层仅负责参数校验和响应 | 🔴 重构 |
| 数据模型 | 增加角色字段、软删除、多边形围栏支持 | 🟡 扩展 |
| 数据库迁移 | 引入 Alembic 管理表结构变更 | 🟢 新增 |
| 后台管理 | 全新开发 Web 管理后台 | 🔴 新增 |
| 围栏可视化 | 集成高德地图实现围栏绘制与管理 | 🔴 新增 |
| 文件管理 | 实现文件上传、存储、访问的完整方案 | 🟢 新增 |
| WebSocket | AI 对话支持流式输出 | 🟢 新增 |
| 运维基础设施 | 日志、限流、健康检查、异常处理 | 🟢 新增 |

### 2.3 兼容性保障原则

- **API 路径兼容**：现有 `/api/*` 路径保持不变，新增接口使用 `/api/v1/*` 前缀
- **响应格式兼容**：现有响应结构不变，新增字段不影响前端解析
- **认证方式兼容**：继续使用 JWT Bearer Token，扩展 Token 中携带角色信息
- **数据库兼容**：表结构变更通过 Alembic 迁移脚本执行，不破坏现有数据

### 4.2 分层架构说明

| 层级 | 职责 | 对应目录 |
|------|------|---------|
| **路由层 (Router)** | 接收请求、参数校验、调用 Service、返回响应 | `app/api/` |
| **服务层 (Service)** | 业务逻辑编排、事务管理、调用 Repository | `app/services/` |
| **数据访问层 (Repository)** | 数据库 CRUD 操作、查询构建 | `app/repositories/` |
| **模型层 (Model)** | ORM 模型定义、数据库表映射 | `app/models/` |
| **模式层 (Schema)** | 请求/响应数据结构定义、数据校验 | `app/schemas/` |
| **核心层 (Core)** | 配置、安全、依赖注入、中间件 | `app/core/` |

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

**密码迁移策略**：旧 SHA256 哈希在用户首次登录时自动升级为 bcrypt，无需强制重置密码。

```python
# app/services/auth.py (密码迁移逻辑)
def authenticate_user(db, username: str, password: str):
    user = get_user_by_username(db, username)
    if not user:
        return None

    if user.password_hash.startswith("$2b$"):
        if not verify_password(password, user.password_hash):
            return None
    else:
        legacy_hash = hashlib.sha256(password.encode()).hexdigest()
        if legacy_hash != user.password_hash:
            return None
        user.password_hash = get_password_hash(password)
        db.commit()

    return user
```

#### 5.3.2 围栏检测优化

**现有方案**：全表扫描 + Python Haversine 计算

**优化方案**：使用 MySQL 空间函数 `ST_Distance_Sphere`

```sql
SELECT *, ST_Distance_Sphere(
    POINT(center_lng, center_lat),
    POINT(#{lng}, #{lat})
) AS distance
FROM geo_fences
WHERE is_active = TRUE
  AND shape_type = 'circle'
HAVING distance <= radius
ORDER BY distance ASC
LIMIT 1;
```

**优化效果**：
- 减少 Python 层计算开销
- 利用 MySQL 空间索引加速查询
- 支持未来扩展到多边形围栏

#### 5.3.3 多边形围栏检测

对于多边形围栏，使用 MySQL 的 `ST_Contains` 函数：

```sql
SELECT * FROM geo_fences
WHERE is_active = TRUE
  AND shape_type = 'polygon'
  AND ST_Contains(
    ST_GeomFromText(polygon_wkt),
    POINT(#{lng}, #{lat})
  );
```

<template>
  <div class="fence-drawer">
    <div class="toolbar">
      <el-radio-group v-model="drawMode">
        <el-radio-button value="circle">圆形围栏</el-radio-button>
        <el-radio-button value="polygon">多边形围栏</el-radio-button>
      </el-radio-group>
    </div>
    <div id="fence-map-container" class="map-container"></div>
  </div>

<script setup>
import AMapLoader from '@amap/amap-jsapi-loader'

let map = null
let circleEditor = null
let polygonEditor = null

AMapLoader.load({
  key: '4e90b4242ed26bcf402a0bb7f46197de',
  version: '2.0',
  plugins: ['AMap.CircleEditor', 'AMap.PolygonEditor', 'AMap.Geocoder']
}).then((AMap) => {
  map = new AMap.Map('fence-map-container', {
    zoom: 15,
    center: [104.065699, 30.580307]
  })
})

function startCircleDraw() {
  const circle = new AMap.Circle({
    center: map.getCenter(),
    radius: 200,
    strokeColor: '#D93025',
    fillColor: '#D93025',
    fillOpacity: 0.15
  })
  map.add(circle)
  circleEditor = new AMap.CircleEditor(map, circle)
  circleEditor.open()
}

function startPolygonDraw() {
  const polygon = new AMap.Polygon({
    strokeColor: '#D93025',
    fillColor: '#D93025',
    fillOpacity: 0.15
  })
  map.add(polygon)
  polygonEditor = new AMap.PolygonEditor(map, polygon)
  polygonEditor.open()
}

function getFenceData() {
  if (drawMode.value === 'circle' && circleEditor) {
    const circle = circleEditor.getTarget()
    const center = circle.getCenter()
    return {
      shape_type: 'circle',
      center_lng: center.lng,
      center_lat: center.lat,
      radius: circle.getRadius()
    }
  }
  if (drawMode.value === 'polygon' && polygonEditor) {
    const polygon = polygonEditor.getTarget()
    const path = polygon.getPath()
    return {
      shape_type: 'polygon',
      polygon_coords: path.map(p => ({ lng: p.lng, lat: p.lat }))
    }
  }
  return null
}

        proxy_set_header X-Real-IP $remote_addr;

## 十一、常见问题排查与解决方案

### 11.1 数据库相关问题

| 问题现象 | 根因分析 | 解决方案 |
|---------|---------|---------|
| **`Table 'xxx.users' doesn't exist`** | 未执行 `init_db.py` 创建表 | ```bash<br>cd NewBackend<br>source .venv/bin/activate<br>python scripts/init_db.py<br>python scripts/seed_data.py<br>``` |
| **MySQL 连接被拒绝** | MySQL 服务未启动 | ```bash<br>sudo systemctl start mysql<br>sudo systemctl status mysql<br>``` |
| **Access denied for user 'root'@'localhost'** | 密码错误或用户不存在 | ```bash<br># 验证 MySQL 登录<br>mysql -u root -p<br># 如忘记密码，重置 MySQL 密码<br>``` |
| **SQLite 模式下围栏检测不准** | SQLite 无空间函数 | 这是预期行为！已自动使用 Python 计算<br>生产环境必须使用 MySQL 8.0 |

### 11.2 后端启动问题

| 问题现象 | 根因分析 | 解决方案 |
|---------|---------|---------|
| **`Connection refused` port 8000** | 后端服务未启动 | 检查 uvicorn 是否在运行<br>查看启动日志是否有报错 |
| **bcrypt 导入错误** | passlib 与 bcrypt 版本不兼容 | 已修复！确保使用最新代码<br>检查 `app/core/security.py` 不依赖 passlib |
| **`SECRET_KEY` 环境变量未设置** | `.env` 配置缺失 | ```bash<br>cd NewBackend<br>cp .env.example .env<br># 编辑 .env 设置 SECRET_KEY<br>``` |
| **CORS 跨域错误** | `ALLOWED_ORIGINS` 未配置 | 在 `.env` 中添加前端域名：<br>`ALLOWED_ORIGINS=http://localhost:3000` |

### 11.3 登录相关问题

| 问题现象 | 根因分析 | 解决方案 |
|---------|---------|---------|
| **登录成功但页面无跳转** | Token 存储问题 | 打开浏览器控制台<br>检查 `localStorage.getItem('token')` |
| **登录提示 "用户名或密码错误"** | 未执行 `seed_data.py` | ```bash<br>python scripts/seed_data.py<br># 验证账号已创建<br>mysql -e "SELECT username,role FROM fanchangjiang_ar.users;"<br>``` |
| **管理员无权限访问后台** | 用户 role 不是 admin | ```bash<br>mysql -e "UPDATE users SET role='admin' WHERE username='admin';"<br>``` |
| **登录后刷新页面又回到登录页** | Token 过期或无效 | 检查 JWT SECRET_KEY 是否一致<br>清除浏览器 localStorage |

### 11.4 前端相关问题

| 问题现象 | 根因分析 | 解决方案 |
|---------|---------|---------|
| **前端 API 请求 404** | Vite 代理配置问题 | 检查 `admin/vite.config.js` 的 proxy 配置<br>确保 `/api` 代理到 `http://localhost:8000` |
| **地图不显示** | 高德地图 Key 缺失 | 在系统配置页面配置 AMAP_KEY<br>或在 `.env` 中设置 |
| **npm install 失败** | Node.js 版本过旧 | ```bash<br>node --version  # 需要 >= 18.x<br># 升级 Node.js<br>curl -fsSL https://deb.nodesource.com/setup_18.x \| sudo -E bash -<br>sudo apt install -y nodejs<br>``` |
