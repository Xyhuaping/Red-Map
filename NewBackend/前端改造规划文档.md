## 一、项目现状分析

### 1.1 现有技术栈问题

| 问题类别       | 现状描述                              | 影响               |
| ---------- | --------------------------------- | ---------------- |
| **框架混用**   | 使用 UniApp 框架，但目标是纯 HTML/CSS/JS 网页 | 架构不清晰，代码冗余       |
| **API 调用** | 使用 `uni.request` 而非标准 Fetch/Axios | 与 Web 标准不一致，移植困难 |
| **样式系统**   | 使用 SCSS 变量但缺少统一设计规范               | 样式碎片化，维护成本高      |
| **状态管理**   | 使用 `ref/reactive` 手动管理，无统一状态层     | 跨组件状态共享困难        |
| **页面路由**   | 使用 UniApp 路由机制                    | 与标准 SPA 路由不一致    |
| **组件化**    | 组件划分不清晰，复用性差                      | 代码重复，维护困难        |

## 二、改造目标

5. **可维护性**：建立统一的代码规范和组件库

## 四、页面设计规范

### 4.1 整体风格

- **设计主题**：红色文化 + 现代简约
- **主色调**：#D93025 (红色)
- **辅助色**：#1A1A1A (深灰), #FFFFFF (白色)
- **背景色**：#F5F7FA (浅灰)
- **圆角规范**：8px (小), 12px (中), 16px (大)
- **阴影规范**：统一使用 `0 2px 8px rgba(0,0,0,0.08)`

### 4.2 页面结构

| 页面  | 路由              | 主要功能               |
| --- | --------------- | ------------------ |
| 首页  | `/`             | 全屏地图 + 位置卡片 + 最近活动 |
| 登录  | `/login`        | 登录/注册表单            |
| 地图  | `/map`          | 围栏展示 + 触发检测        |
| 人物  | `/figures`      | 人物列表 + 搜索          |
| 对话  | `/chat?id={id}` | AI 对话界面            |
| 统计  | `/stats`        | 数据统计 + 轨迹列表        |
| 个人  | `/profile`      | 用户信息 + 菜单          |

### 4.3 响应式策略

- **移动优先**：默认设计针对移动端
- **断点设置**：
  - `< 768px`: 手机端
  - `768px - 1024px`: 平板端
  - `> 1024px`: 桌面端
- **适配方式**：CSS Grid + Flexbox + Media Queries

## 六、功能模块详细设计

### 6.1 地图模块 (Map)

**核心功能**：

- 高德地图加载与初始化
- 用户位置获取与显示
- 围栏覆盖物渲染
- 标记点展示
- 围栏触发检测

**关键实现**：

```javascript
// 地图初始化
function initMap(container, options) {
  return new Promise((resolve, reject) => {
    AMap.init(() => {
      const map = new AMap.Map(container, options);
      resolve(map);
    }, reject);
  });
}

// 围栏检测
function checkFenceTrigger(userLoc, fences) {
  return fences.filter(fence => {
    const distance = calculateDistance(
      userLoc.lng, userLoc.lat,
      fence.center_lng, fence.center_lat
    );
    return distance <= fence.radius;
  });
}
```

### 6.2 认证模块 (Auth)

**核心功能**：

- 用户注册
- 用户登录
- Token 管理
- 登录状态维护

**关键实现**：

```javascript
// Token 管理
const TokenManager = {
  get() {
    return localStorage.getItem('token');
  },
  set(token) {
    localStorage.setItem('token', token);
  },
  remove() {
    localStorage.removeItem('token');
  }
};
```

### 6.3 对话模块 (Chat)

**核心功能**：

- 消息列表渲染
- 消息发送与接收
- 对话历史加载
- 打字机效果

**关键实现**：

```javascript
// 消息渲染
function renderMessage(msg, container) {
  const div = document.createElement('div');
  div.className = `message ${msg.role}`;
  div.innerHTML = `
    <div class="avatar">${getAvatar(msg.role)}</div>
    <div class="content-box">
      <div class="content">${escapeHtml(msg.content)}</div>
      <div class="time">${formatTime(msg.time)}</div>
    </div>
  `;
  container.appendChild(div);
}
```

### 6.4 统计模块 (Stats)

**核心功能**：

- 统计数据获取
- 统计卡片展示
- 轨迹列表展示
- 进度条渲染

**关键实现**：

```javascript
// 统计计算
function calculateStats(tracks) {
  return {
    total_visits: tracks.length,
    unique_figures: new Set(tracks.map(t => t.figure_id)).size,
    total_duration: tracks.reduce((sum, t) => sum + (t.duration || 0), 0),
    total_rounds: tracks.reduce((sum, t) => sum + (t.rounds || 0), 0)
  };
}
```
